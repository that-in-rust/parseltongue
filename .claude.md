## Rule #2: NO STUBS, NO PLACEHOLDERS, NO INCOMPLETE CODE

**Absolutely forbidden:**
- ‚ùå Committing code with "TODO:" comments
- ‚ùå Committing code with "STUB:" markers
- ‚ùå Committing code with "PLACEHOLDER:" comments

- ‚ùå Shipping releases with incomplete implementations
- ‚ùå Leaving open todos in commits
- ‚ùå Creating "skeleton" implementations that don't actually work

**Required behavior:**
- ‚úÖ Either implement it FULLY or DON'T commit it
- ‚úÖ Every commit should be production-ready
- ‚úÖ Every release should have ZERO stubs
- ‚úÖ If you can't implement something fully, SAY SO explicitly
- ‚úÖ Remove ALL todos before marking work complete
- ‚úÖ Test that everything actually works before committing

**If you can't complete something:**
- Say: "I cannot complete this fully right now because [reason]"
- Ask: "Do you want me to implement X and Y but leave Z for later?"
- Don't just ship incomplete garbage

## Rule #3: NO OPEN TODOS IN COMMITS

**The rule:**
- Before ANY commit, ALL todos must be marked complete
- If there are open todos, the work is NOT DONE
- Don't commit work-in-progress
- Don't ship releases with incomplete task lists

**If work is incomplete:**
- Explicitly tell the user: "This is not ready to commit because [open todos]"
- Ask if they want you to finish or if they want to handle it differently
